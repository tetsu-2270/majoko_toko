# 8. シーケンス設計

## 8.1 概要

本章では、アプリケーション起動からWordPressへの投稿完了までの処理シーケンスを定義する。

------------------------------------------------------------------------

## 8.2 メインシーケンス

``` text
User
 │
 │ 実行
 ▼
Application
 │
 ├─ ConfigManager : 設定読込
 ├─ GmailClient : 未読メール取得
 ├─ AttachmentManager : 添付画像保存
 ├─ ImageSorter : 画像並び替え
 ├─ HtmlGenerator : HTML生成
 ├─ WordPressClient : 画像アップロード
 ├─ WordPressClient : カテゴリ取得・作成
 ├─ WordPressClient : 記事投稿
 ├─ HistoryManager : 投稿履歴保存
 └─ LogManager : 完了ログ
```

------------------------------------------------------------------------

## 8.3 Gmail通信

``` text
Application
      │
      ▼
 Gmail Server
      │
 未読メール取得
      │
 添付画像取得
      │
 Message-ID取得
      │
 ラベル付与
      │
 既読化
```

------------------------------------------------------------------------

## 8.4 WordPress通信

``` text
Application
      │
      ▼
 WordPress REST API
      │
 Media Upload
      │
 Category Search
      │
 Category Create
      │
 Post Create
      │
 Featured Image Set
      │
 URL取得
```

------------------------------------------------------------------------

## 8.5 エラーシーケンス

``` text
API実行
   │
   ├──成功
   │
   ▼
 次処理
   │
   └──失敗
        │
        ▼
   Retry(最大3回)
        │
        ├─成功 → 次処理
        │
        └─失敗
              │
              ▼
         ERRORログ出力
              │
              ▼
            終了
```

------------------------------------------------------------------------

## 8.6 ログ保存タイミング

以下のイベントでログを出力する。

-   アプリ起動
-   Gmail接続
-   メール取得
-   添付保存
-   画像アップロード
-   記事投稿
-   投稿完了
-   エラー
-   終了

------------------------------------------------------------------------

## 8.7 履歴保存

投稿成功時のみ保存する。

保存内容

-   投稿ID
-   投稿URL
-   投稿日時
-   Message-ID
-   カテゴリ
-   タイトル
