# 8. シーケンス設計

## 8.1 概要

本章では、アプリケーション起動からWordPressへの投稿完了までの処理シーケンスを定義する。

------------------------------------------------------------------------

## 8.2 メインシーケンス

``` text
User
 │
 │ 実行
 ▼
Application
 │
 ├─ ConfigManager : 設定読込
 ├─ GmailClient : 未読メール取得（各メールのinternalDateを含む）
 ├─ Application : 受信時刻の古い順へ安定ソート・処理順をログ記録
 │
 │  【以下、メール1件ずつ直列に処理（受信時刻順）】
 │
 ├─ Application : 話数解決（明示話数、または WordPressClient.find_latest_post_number
 │                 と実行中キャッシュの最大値+1で自動採番）
 ├─ AttachmentManager : 添付画像保存
 ├─ ImageSorter : 画像並び替え
 ├─ WordPressClient : 画像アップロード
 ├─ WordPressClient : カテゴリ取得・作成
 ├─ Application : 前回記事解決（実行中キャッシュ→WordPress標準形式→旧形式、
 │                 同一作品の直前話のみ。無ければ前回セクションなし）
 ├─ HtmlGenerator : HTML生成
 ├─ WordPressClient : 記事投稿
 ├─ Application : 投稿成功記事を実行中キャッシュへ登録
 ├─ HistoryManager : 投稿履歴保存
 │
 │  【次のメールへ（失敗時も受信時刻順のまま継続）】
 │
 └─ LogManager : 完了ログ
```

------------------------------------------------------------------------

## 8.5 エラーシーケンス

``` text
API実行
   │
   ├──成功
   │
   ▼
 次処理
   │
   └──失敗
        │
        ▼
   Retry(最大3回)
        │
        ├─成功 → 次処理
        │
        └─失敗
              │
              ▼
         ERRORログ出力
              │
              ▼
            終了
```

------------------------------------------------------------------------

## 8.6 ログ保存タイミング

以下のイベントでログを出力する。

-   アプリ起動
-   Gmail接続
-   メール取得
-   複数メールの処理順確定（受信時刻・件名・Message-ID）
-   添付保存
-   画像アップロード
-   前回記事の解決結果（実行中キャッシュ／WordPress標準形式／旧形式／前回記事なし）
-   記事投稿
-   投稿完了
-   エラー
-   終了

本文・添付内容・認証情報はログへ出力しない。

------------------------------------------------------------------------

## 8.7 履歴保存

投稿成功時のみ保存する。

保存内容

-   投稿ID
-   投稿URL
-   投稿日時
-   Message-ID
-   カテゴリ
-   タイトル
