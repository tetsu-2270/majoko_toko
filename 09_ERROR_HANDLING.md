# 9. エラー処理設計

## 9.1 目的

本システムで発生し得るエラーを分類し、検知方法・対処方法・ログ出力方針を定義する。

------------------------------------------------------------------------

## 9.2 エラー一覧

  ID        分類             内容                   対応
  --------- ---------------- ---------------------- -----------------
  ERR-001   設定             config.yaml 読込失敗   起動中止
  ERR-002   認証             Gmail認証失敗          終了・ERRORログ
  ERR-003   通信             IMAP接続失敗           3回リトライ
  ERR-004   入力             添付画像なし           スキップ
  ERR-005   画像             画像保存失敗           ERRORログ
  ERR-006   API              WordPress認証失敗      終了
  ERR-007   API              画像アップロード失敗   3回リトライ
  ERR-008   API              記事投稿失敗           3回リトライ
  ERR-009   テンプレート     template.html 不在     終了
  ERR-010   予期しない例外   未捕捉例外             ログ出力後終了

------------------------------------------------------------------------

## 9.3 リトライポリシー

対象: - 通信タイムアウト - HTTP 429 - 一時的なHTTP 500系

設定: - 最大3回 - 指数バックオフ - 失敗時はERRORログ出力

------------------------------------------------------------------------

## 9.4 ログレベル

  レベル   用途
  -------- --------------------
  INFO     通常処理
  WARN     継続可能な異常
  ERROR    処理中断を伴う異常
  DEBUG    開発時のみ

------------------------------------------------------------------------

## 9.5 異常終了条件

-   設定ファイルが読めない
-   Gmail認証不可
-   WordPress認証不可
-   テンプレート読込不可
-   リトライ上限到達

------------------------------------------------------------------------

## 9.6 運用手順

### Gmailエラー

-   アプリケーションパスワード確認
-   IMAP有効化確認

### WordPressエラー

-   Application Password確認
-   REST API疎通確認

### テンプレートエラー

-   template.html 配置確認

------------------------------------------------------------------------
