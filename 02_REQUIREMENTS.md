# 02_REQUIREMENTS.md

# 要件定義

## 1. 目的

Gmailへ送信されたメールを自動解析し、WordPressへ記事を自動投稿する。

------------------------------------------------------------------------

# 2. 機能要件

## 2.1 Gmail受信

-   未読メールのみ取得
-   「投稿済み」ラベルが付いていないメールのみ対象
-   Gmail APIを利用する
-   Gmail APIが返す`internalDate`（受信時刻epochミリ秒）を、そのメールの受信時刻の
    正本として取得する。取得できない場合のみメールの`Date`ヘッダーへフォールバックし、
    どちらも得られない場合は受信時刻不明としてアプリを停止せず処理を続ける。

## 2.2 件名解析

推奨件名形式

    作品名 ~ 話数

例

    お菓子外しさんとカタカナ男 ~ 42

以下の表記揺れも話数として認識し、標準形式へ正規化する。

    作品名 42
    作品名　４２（全角スペース・全角数字）
    作品名 ~ 42
    作品名　〜　４２（全角スペース・波ダッシュ・全角数字）
    作品名 No42（旧No形式。No/no・Ｎｏ／ｎｏ、No後のスペース有無を問わない）
    作品名 No 42
    作品名 Ｎｏ４２

### 解析の優先順位

件名末尾の話数表記は、次の優先順で判定する。

1.  `~`・`〜`・`～`区切りの明示話数
2.  半角/全角`No`形式の明示話数
3.  区切りなし（空白のみ）の裸の数字

この順で判定することにより、`作品名 No 42`のような「No」区切りが「区切りなしの
裸の数字」として誤って解釈され、`No`まで作品名に取り込まれてしまうことを防ぐ
（誤: 作品名`作品名 No`・話数42。正: 作品名`作品名`・話数42）。

### ルール

-   作品名部分を記事タイトルの元として利用する。
-   話数として扱うのは件名末尾の独立した数字のみ（作品名途中の数字は話数として扱わない）。
-   **件名に話数が無い場合は、件名全体を作品名として扱い、WordPress上の同一作品名の最新記事を検索して最大話数+1を自動採番する。既存記事が1件も無い場合は第1話とする。**
-   話数解決（明示話数の使用、または自動採番）は、画像アップロード・カテゴリ作成より前に行う。最新話数検索に失敗した場合、WordPressへ画像やカテゴリを作成しない。
-   WordPress上の投稿タイトルは、話数の記載形式・自動採番のいずれであっても必ず次の標準形式で作成する。

        作品名 ~ 話数

-   空の件名は無効としてスキップし、理由をWARNログへ記録する。「作品名 ~」「作品名 No」のように区切り表記だけが残り話数が存在しない件名も同様に無効とする。

## 2.2.1 複数メールの処理順・自動採番・前回リンク

1回の実行で複数の投稿メールを受信した場合、以下を満たす。

-   Gmailの一覧返却順を信頼せず、`internalDate`（受信時刻）が古いメールから順に
    1件ずつ直列で処理する（並列投稿は行わない）。
-   話数省略メールの自動採番も、この受信時刻順に行う。
-   同一実行中に投稿成功した記事は、実行中だけ保持するキャッシュ（作品名・話数の
    組で管理）へ登録し、後続メールの自動採番・前回記事解決から参照できるようにする
    （WordPress側の検索結果が直後の投稿をまだ反映していない場合があるため）。
    投稿失敗・重複履歴でスキップした記事はキャッシュへ登録しない。
-   前回リンクは、同一作品名の直前話（現在話数-1）だけを対象とする。別作品や
    同一カテゴリの最新記事、部分一致した記事を前回リンクとして採用しない。
-   直前話が存在しない場合（第1話、または欠番）は、前回セクションを一切生成しない。
    直前話が無いことを理由に、それより前の話数へ遡ることはしない。
-   複数作品が同一実行に混在しても、前回リンクが作品間で交差しないようにする。

## 2.3 メール本文

本文形式

    カテゴリー：4コマ

    投稿日：2026/07/10 20:00

### 必須

-   カテゴリー

### 任意

-   投稿日（未指定なら即時公開）

## 2.4 添付画像

-   添付画像をimagesフォルダへ保存
-   ファイル名は変更しない
-   IMG9999→IMG0001を考慮して並び替える
-   並び替え後の順番でWordPressへアップロード

## 2.5 HTML生成

template.htmlを読み込む。

### 前回記事

    <!-- 前回の話 -->

を前回記事リンクへ置換する。

### 本文

    <!-- 以下本文 -->

を画像HTMLへ置換する。

## 2.6 WordPress

-   Application Password認証
-   REST API利用
-   カテゴリーが存在しない場合は自動作成
-   1枚目画像をアイキャッチ画像に設定
-   投稿日指定時は予約投稿

## 2.7 投稿成功後

成功時のみ

-   Gmail既読
-   投稿済みラベル付与
-   投稿履歴DB登録

失敗時は何も変更しない。

------------------------------------------------------------------------

# 3. 非機能要件

-   Python3.12以上
-   SQLite利用
-   logging利用
-   型ヒント必須
-   pytest対応
-   1モジュール1責務

# 4. 除外事項

初期バージョンでは対象外

-   AI文章生成
-   AI画像生成
-   SNS同時投稿
-   GUI
-   Docker
