# 4. 処理フロー設計

## 4.1 全体フロー

``` text
開始
 ↓
Gmail APIで未読メール取得
 ↓
対象メール判定
 ↓
添付画像取得
 ↓
画像番号順へ並び替え
 ↓
WordPressへ画像アップロード
 ↓
カテゴリ確認・作成
 ↓
記事番号採番
 ↓
HTMLテンプレート生成
 ↓
アイキャッチ設定
 ↓
記事投稿
 ↓
履歴保存
 ↓
メールへラベル付与・既読化
 ↓
終了
```

## 4.2 Gmail受信

-   アプリケーションパスワードを利用してIMAP接続
-   未読メールのみ取得
-   添付画像が存在するメールのみ対象
-   処理済みラベルが付いたメールは除外

## 4.3 添付画像取得

-   JPG / JPEG / PNG対応
-   一時フォルダへ保存
-   EXIFは保持

## 4.4 並び替え

ファイル名を基準に昇順ソート。

例

    IMG9998
    IMG9999
    IMG0001
    IMG0002

日付を考慮して

    IMG9998
    IMG9999
    IMG0001
    IMG0002

の順で投稿する。

## 4.5 WordPressアップロード

-   元サイズ画像をアップロード
-   リサイズなし
-   Media API使用

## 4.6 HTML生成

template.htmlへ

-   タイトル
-   本文
-   画像HTML

を差し込み生成。

## 4.7 投稿

-   下書き保存
-   アイキャッチ設定
-   カテゴリ設定
-   タグ設定（将来拡張）

## 4.8 エラー処理

  内容                   動作
  ---------------------- ----------
  認証失敗               終了
  画像アップロード失敗   リトライ
  投稿失敗               ログ保存
  テンプレートなし       終了

## 4.9 ログ

記録内容

-   開始日時
-   終了日時
-   投稿ID
-   投稿URL
-   エラー内容

## 4.10 シーケンス概要

``` text
User
 │
 │実行
 ▼
Application
 │
 ├─ Gmail
 ├─ Download Images
 ├─ Sort Images
 ├─ Upload WP
 ├─ Generate HTML
 ├─ Create Post
 └─ Save Log
```
