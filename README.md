# majoko_toko

Gmailで受信した画像付き投稿メールを解析し、添付画像をWordPressへアップロードして
記事（下書き／予約投稿）を自動作成するアプリケーション。

## 投稿メールの形式

-   件名（推奨形式）: `作品名 ~ 話数`（例: `お菓子外しさんとカタカナ男 ~ 41`）
    -   `作品名 41` / `作品名　４２`（全角スペース・全角数字） / `作品名　〜　４２`（波ダッシュ） /
        `作品名 No42` / `作品名 No 42` / `作品名 Ｎｏ４２`（旧No形式。半角/全角、大文字/小文字、
        No後のスペース有無を問わない）などの表記揺れも話数として認識し、標準形式
        `作品名 ~ 話数`へ正規化して投稿する（`No`区切りが誤って作品名へ取り込まれることはない）
    -   **話数を省略した場合**は件名全体を作品名として扱い、WordPress上の同一作品名の
        最新記事から最大話数+1を自動採番する（既存記事が無ければ第1話）
    -   空の件名、または`作品名 ~`・`作品名 No`のように区切り表記だけが残り話数が存在しない
        件名は無効として投稿せずスキップし、理由をログへ記録する
-   本文:
    -   `カテゴリ：{カテゴリ名}`（`カテゴリー：` 表記も可）
    -   `投稿日：{yyyy/mm/dd hh:mm}`（省略時は下書き保存）
-   添付ファイル: JPG / JPEG / PNG（大文字・小文字は区別しない）。
    ファイル名は `IMG9999→IMG0001` のようなロールオーバーを考慮して並び替えたうえで
    アップロードする。対応形式以外の添付は無視する。対応画像が1枚も無いメールは
    投稿を行わずスキップする。

## 起動方法

```bash
python3 main.py
```

`config/config.yaml` に基づき、未読メールの取得からWordPress投稿・履歴保存までを
1回実行する。継続実行する場合はOS側のスケジューラ等で本コマンドを定期実行する。

複数の投稿メールが同時に届いた場合は、Gmailの受信時刻（`internalDate`）が古い順に
1件ずつ直列で投稿する（一覧の返却順やAPI呼び出し順は信頼しない）。話数省略メールの
自動採番もこの順序で行い、同一実行中に投稿した記事は前回リンク・自動採番の対象に
含める（次回起動へは引き継がない）。前回リンクは同一作品の直前話だけを対象とし、
直前話が無い場合（第1話・欠番）は前回セクション自体を表示しない。

## 確認方法

-   ログ: `config/config.yaml` の `paths.log_file` に出力される
-   投稿履歴: `paths.history_file`（メールの重複投稿防止に使用）
