# 12. 運用手順書

## 12.1 目的

本書は、アプリケーションの日常運用、障害発生時の対応、および定期メンテナンス手順を定義する。

------------------------------------------------------------------------

## 12.2 運用前チェック

-   config.yaml が配置されている
-   template.html が存在する
-   config/credentials.json が配置されている
-   初回のGoogleログイン・スコープ同意が完了し config/token.json が生成されている
-   WordPress Application Password 設定済み
-   ネットワーク疎通確認

------------------------------------------------------------------------

## 12.3 起動手順

``` bash
python main.py
```

確認事項

-   エラーが表示されない
-   ログが生成される
-   Gmailへ接続できる

------------------------------------------------------------------------

## 12.4 通常運用フロー

1.  Gmail未読メール取得
2.  添付画像取得
3.  画像並び替え
4.  WordPressへ画像アップロード
5.  記事作成（下書き）
6.  投稿履歴保存
7.  メールへラベル付与・既読化

------------------------------------------------------------------------

## 12.5 障害対応

  障害                   確認事項                        対応
  ---------------------- ------------------------------- ----------
  Gmail接続不可          credentials.json・token.json    設定修正・再認証
  WordPress認証失敗      Application Password            再発行
  画像アップロード失敗   API応答                         再実行
  テンプレートなし       ファイル有無                     配置

------------------------------------------------------------------------

## 12.6 ログ確認

ログファイル:

    logs/application.log

確認ポイント

-   ERROR
-   WARN
-   Retry回数
-   投稿ID

------------------------------------------------------------------------

## 12.7 定期メンテナンス

-   ログローテーション
-   投稿履歴バックアップ
-   テンプレート更新
-   パスワード更新
-   Pythonライブラリアップデート

------------------------------------------------------------------------

## 12.8 バックアップ

対象

-   config.yaml
-   template.html
-   token.json（再認証の手間を省くため。ただし取り扱いは機密情報に準じる）
-   post_history.json
-   logs

対象外（バックアップ・共有ともに不可、漏洩時は再発行）

-   credentials.json

推奨頻度

-   毎日
-   更新前

------------------------------------------------------------------------

## 12.9 運用上の注意

-   認証情報をGitへ登録しない
-   ログへパスワードを書き込まない
-   大量投稿前はテスト実施
-   本番は下書き投稿で確認後に公開
