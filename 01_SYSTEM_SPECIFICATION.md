# 1. システム概要

## 1.1 システム名

Majokko Auto Publisher（略称：MAP）

## 1.2 システム目的

Gmailで受信した投稿メールを解析し、WordPress REST
APIを利用して記事を自動生成・公開する。

記事投稿時に必要となる

-   タイトル採番
-   画像アップロード
-   サムネイル設定
-   カテゴリ作成
-   前回記事リンク生成
-   予約投稿

を完全自動化する。

------------------------------------------------------------------------

# 2. システム構成

MAPは以下のサブシステムで構成される。

-   Configuration
-   Workflow Engine
-   Gmail Service
-   Mail Parser
-   Image Service
-   WordPress Service
-   Template Engine
-   Database
-   Logging
-   Scheduler

各サブシステムは独立して動作する。
サブシステム間はDataModelのみを共有する。

# 3. 開発目的

-   投稿作業時間の削減
-   投稿品質の統一
-   人的ミス防止
-   長期運用
-   AIによる保守

# 4. 基本機能

1.  Gmail取得
2.  メール解析
3.  添付画像保存
4.  添付画像並び替え
5.  WordPress画像アップロード
6.  カテゴリ取得・作成
7.  タイトル採番
8.  前回記事検索
9.  HTML生成
10. サムネイル設定
11. 投稿
12. 予約投稿
13. 投稿履歴保存
14. Gmail既読化
15. Gmailラベル付与
16. ログ保存

------------------------------------------------------------------------

# 5. 非機能要件

-   1モジュール1責務
-   型ヒント必須
-   docstring必須
-   投稿失敗時はロールバック
-   Gmail・WordPressの交換可能な構造

------------------------------------------------------------------------

# 6. 対応環境

-   macOS
-   Windows 11
-   Python 3.12以上（推奨）
-   WordPress REST API
-   Gmail API OAuth

------------------------------------------------------------------------

# 7. 設計原則

1.  1モジュール1責務
2.  設定値はConfigのみ
3.  固定値禁止
4.  ログ必須
5.  例外処理必須
6.  型ヒント必須
7.  DataModel経由で受け渡し
8.  Workflowが全体制御
